La API aplica límites de tasa por dirección IP para proteger el servicio y mantener una experiencia consistente para todos los integradores.

## Endpoints públicos de consulta

Los endpoints públicos (`/productos/{licenseKey}`, `/estaciones/{licenseKey}`) están protegidos por una política de tasa por IP. Si superas el límite recibes una respuesta **HTTP 429 (Too Many Requests)**.

Los límites concretos se ajustan según el plan contratado y según la carga global del servicio. Si tu integración necesita un volumen consistentemente alto, escríbenos a [soporte@wrestaurantapi.com](mailto:soporte@wrestaurantapi.com) para discutir un plan dedicado con límites personalizados.

## Qué pasa al exceder el límite

La respuesta es `HTTP 429`. La API incluye el encabezado estándar `Retry-After` indicando cuántos segundos debes esperar antes de reintentar.

## Buenas prácticas

- Implementa **back-off exponencial** ante 429. Empieza con 1 segundo, duplica hasta un máximo de 30 segundos.
- Respeta el encabezado `Retry-After` cuando esté presente; te dice exactamente cuándo el contador se libera.
- Cachea respuestas que cambien poco — los catálogos de productos y las listas de estaciones son buenos candidatos.
- Distribuye las llamadas en el tiempo en vez de ejecutarlas en ráfagas: un goteo constante consume menos cuota que picos cortos.
- Una sola API key por entorno (producción vs. staging) hace más sencillo aislar tráfico ruidoso.
