Cada **sucursal** que conectes recibe una **license key** única que la identifica en todas las llamadas a la API.

## Formato

16 caracteres alfanuméricos, ejemplo:

```
DS5D2368HCME4O9X
```

## Cómo se obtiene

La license key se genera al instalar el componente cliente en el equipo de la sucursal. Aparece en tu panel bajo la sección **Sucursales**.

## Cómo se usa

Todas las llamadas a endpoints de datos incluyen la license key en la ruta:

```
GET /api/v0/productos/{licenseKey}
GET /api/v0/estaciones/{licenseKey}
```

La API valida que la license key:

1. **Existe** y está activa.
2. **Pertenece a tu cuenta** (la API key que envías debe ser dueña de la sucursal).
3. **Está en línea** (la sucursal responde en este momento).

Si cualquiera de esas condiciones falla, recibes un error 4xx con un mensaje claro. Ver [Errores](/errores).

## Múltiples sucursales

Una cuenta puede tener N sucursales, cada una con su propia license key. Las llamadas son por sucursal — no hay endpoints que agreguen datos entre varias sucursales. Si necesitas eso, agrégalos en tu integración.
