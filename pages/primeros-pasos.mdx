Esta guía te lleva de cero a tu primera respuesta exitosa en menos de cinco minutos.

## 1. Crea una cuenta y un API key

1. Regístrate en [https://app.wrestaurantapi.com](https://app.wrestaurantapi.com).
2. En tu panel, ve a **API Keys → Crear nueva**.
3. **Copia la clave inmediatamente** — solo se muestra una vez. Formato: `wrst_live_` seguido de 32 caracteres alfanuméricos.

## 2. Conecta una sucursal

Cada sucursal con Soft Restaurant necesita instalar el componente cliente de W Restaurant para quedar disponible en la API. Al instalarlo, la sucursal recibe una **license key** de 16 caracteres (ejemplo: `DS5D2368HCME4O9X`). Esa license key identifica a cada sucursal en las llamadas. Ver [Sucursales](/workers) para detalles.

## 3. Haz tu primera llamada

```bash
curl -H "X-Api-Key: wrst_live_TU_LLAVE_AQUI" \
  https://wrestaurant-api.inowu.store/api/v0/productos/DS5D2368HCME4O9X?page=1&limit=10
```

Respuesta esperada:

```json
{
  "items": [
    { "id": "...", "name": "Tacos al pastor", "price": 95.00 }
  ],
  "page": 1,
  "limit": 10,
  "totalCount": 142
}
```

## 4. Lo que sigue

- Lee [Autenticación](/autenticacion) para entender el modelo de claves.
- Revisa [Errores](/errores) para manejar respuestas no exitosas.
