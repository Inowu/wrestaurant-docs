# Autenticación

Toda llamada a la API pública usa **API keys** vía el encabezado `X-Api-Key`.

## Formato de la clave

```
wrst_live_<32 caracteres alfanuméricos>
```

Total: 40 caracteres.

## Cómo usar la clave

Incluye el encabezado `X-Api-Key` en cada solicitud:

```bash
curl -H "X-Api-Key: wrst_live_Abc123Def456Ghi789Jkl012Mno345Pq" \
  https://wrestaurant-api.inowu.store/api/v0/productos/DS5D2368HCME4O9X
```

## Cómo se almacena

Almacenamos solo el hash SHA-256 de tu clave junto con sus primeros 16 caracteres (prefix) para identificación visual. La clave completa **nunca se guarda** en texto plano y solo se muestra una vez al crearla. Si la pierdes, debes generar una nueva.

## Gestión de claves

- **Crear:** desde el panel de administración (`POST /api/v0/apikeys`, autenticado con JWT).
- **Listar:** muestra prefijos, fechas de creación y uso reciente (`GET /api/v0/apikeys`).
- **Revocar:** invalida la clave inmediatamente (`DELETE /api/v0/apikeys/{id}`).

## Buenas prácticas

- **Nunca** comprometas API keys a un repositorio público. Usa variables de entorno o un gestor de secretos.
- Rota claves periódicamente (cada 90 días es razonable).
- Crea una clave por entorno (producción, staging) — facilita la rotación y la auditoría.
- Revoca de inmediato si sospechas exposición.

## Límites por clave

Cada API key respeta los [límites de tasa](/limites) globales. No hay cuotas por clave individual en el plan actual.
